# Postman Testing Guide

This guide explains how to test the University Library & Financial Management System API using Postman.

## 1. Prerequisites
- **Postman** installed on your machine.
- **Backend running**: Ensure your Spring Boot backend is running on `http://localhost:8080`.
- **Database ready**: Ensure PostgreSQL is running and the `library_db` is initialized.

## 2. Import the Collection
1. Open Postman.
2. Click **Import** (top left).
3. Drag and drop the file `university_library_api.postman_collection.json` (located in the project root) into Postman.
4. Click **Import** to confirm.

## 3. How to Authenticate
The API uses JWT (JSON Web Tokens) for security.
1. In the imported collection, open the **Auth** folder.
2. Select the **Login** request.
3. The body is already pre-filled with the default admin credentials:
   - **Username**: `admin`
   - **Password**: `admin123`
4. Click **Send**.
5. Upon successful login, the `jwtToken` environment variable will be automatically updated by a script in the "Tests" tab.
6. All other requests in the collection are configured to use this token automatically.

## 4. Testing Endpoints
Once logged in, you can test other folders:
- **Dashboard**: Get system-wide statistics.
- **Books**: List, search, and create books.
- **Members**: Manage students and staff.
- **Borrowing**: Process loans and returns.

## 5. Troubleshooting
- **401 Unauthorized**: Make sure you have run the **Login** request successfully.
- **Connection Refused**: Check if your backend is running on port `8080`.
- **403 Forbidden**: Some actions (like deleting books) require the `ADMIN` role.
